# Cline 사용자 매뉴얼

## 1. 개요 및 개발사
- **도구명**: Cline (클라인)
- **개발사**: Open-source 커뮤니티 (이전 Devins 프로젝트 등에서 파생)
- **한 줄 요약**: VS Code 내에서 동작하는 초강력 오픈소스 에이전트 확장 프로그램으로, 파일 읽기/쓰기, 터미널 실행, 브라우저 스크린샷 캡처 등을 유기적으로 수행합니다.
- **주요 키워드**: `#통합개발환경`, `#오픈소스`, `#에이전트루프`, `#터미널제어`

---

## 2. 핵심 기능 및 AI 에이전트 기능
- **System Command Execution**: 사용자의 승인하에 로컬 터미널 명령어를 직접 수행하여 패키지를 설치하거나 빌드 오류를 직접 바로잡습니다.
- **Incremental File Editing**: 파일 전체를 덮어쓰지 않고 변경이 필요한 세부 블록 단위로 코드를 정밀하게 교체(Patch)하여 토큰을 절약하고 안전하게 편집합니다.
- **Interactive Approval Model**: 에이전트가 어떤 툴(터미널 명령어 실행, 파일 쓰기 등)을 호출할 때마다 매번 사용자에게 허가(Approve)를 요청하여 통제권을 잃지 않게 제어합니다.
- **Local/Cloud LLM Support**: OpenRouter, Anthropic Claude, OpenAI, Ollama(로컬 LLM) 등 수많은 백엔드 API와 호환됩니다.

---

## 3. 설치 및 설정 방법
1. VS Code 확장 프로그램 마켓플레이스에서 'Cline'을 검색하여 설치합니다.
2. 사이드바에 나타나는 Cline 로봇 아이콘을 누르고 설정 단추를 클릭합니다.
3. 원하는 LLM 프로바이더를 선택하고 API Key를 입력합니다. (예: OpenRouter -> Anthropic Claude 3.5 Sonnet 조합이 가장 권장됨)

---

## 4. 핵심 사용 명령어 및 워크플로우
- **태스크 지시**: 사이드바 입력 콘솔에 작업 내용을 씁니다. (예: "이 Node.js 프로젝트의 모든 API에 로깅 미들웨어를 심고 테스트가 돌아가는지 보여줘.")
- **도구 실행 승인**: Cline이 작업을 시작하면 단계별로 수행하려는 도구 액션(예: `execute_command: npm run test`)이 노출되며, 아래의 'Approve' 버튼을 누르면 작업이 이어나갑니다.
- **중단 및 교정**: 에이전트가 오작동하거나 엉뚱한 루프에 돌입한 경우, 언제든 'Stop'을 눌러 흐름을 중단하고 대화창을 통해 지시사항을 정정합니다.

---

## 5. 고급 팁 및 에이전틱 활용 방법
- **Ollama 연동 로컬 사용**: 인터넷이 연결되지 않은 환경에서도 로컬 컴퓨터에 Ollama를 띄우고 `Llama-3`나 `Qwen-2.5-Coder` 모델을 연결하여 기밀 유지가 필요한 코딩 작업을 완전 자율 에이전트에게 맡길 수 있습니다.
- **자동 승인(Auto-approve) 모드**: 매번 클릭하기 번거로운 특정 명령어(예: 파일 읽기 등)는 설정에서 자동 승인 옵션을 활성화하여 극도로 빠르게 속도를 단축할 수 있습니다.
