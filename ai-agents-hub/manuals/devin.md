# Devin 사용자 매뉴얼

## 1. 개요 및 개발사
- **도구명**: Devin (데빈)
- **개발사**: Cognition (코그니션)
- **한 줄 요약**: 개발자 없이도 쉘, 브라우저, 에디터가 탑재된 가상 샌드박스 내부에서 스스로 문제를 진단하고 소프트웨어를 완성하는 최초의 자율형 AI 소프트웨어 엔지니어입니다.
- **주요 키워드**: `#상용서비스`, `#에이전트루프`, `#터미널제어`, `#브라우저제어`

---

## 2. 핵심 기능 및 AI 에이전트 기능
- **샌드박스 코딩 환경**: Devin은 가상의 도커(Docker) 컨테이너 내부에서 안전하게 코드를 컴파일하고 실행합니다.
- **자율 디버깅 및 테스팅**: 코드 실행 후 터미널에 발생하는 오류 메시지를 읽고, 코드를 역추적하여 수정한 다음 재시험하는 루프를 자동으로 진행합니다.
- **웹 리서치 (Web Browser)**: 최신 외부 API나 프레임워크를 학습하기 위해 웹사이트 검색 및 공식 기술 문서를 서핑하여 솔루션을 파악합니다.
- **실시간 진행 상황 대시보드**: 기획(Planning), 실행 로그, 파일 트리 변경 이력을 전용 브라우저 웹 UI로 사용자에게 투명하게 공유합니다.

---

## 3. 설치 및 설정 방법
1. Devin은 로컬 머신에 직접 설치하지 않고, Cognition에서 제공하는 클라우드 기반 웹 플랫폼에 접속하여 사용합니다.
2. 깃허브(GitHub) 저장소 권한을 연동하여 작업할 레포지토리를 활성화합니다.
3. Devin에게 작업을 부여할 때 필요한 API 비밀키(Secret keys)나 토큰값은 대시보드 내 환경 변수 설정란에 안전하게 등록합니다.

---

## 4. 핵심 사용 명령어 및 워크플로우
- **프롬프트 입력**: 웹 UI의 입력창에 자연어로 목표를 제시합니다. (예: "우리 리액트 앱에 라이브 대시보드를 추가하고 로컬호스트 3000포트에서 테스트해줘.")
- **진행 상황 모니터링**: Devin이 세운 Step-by-Step 계획을 보고, 중간에 틀린 방향으로 가면 실시간 피드백 입력창을 통해 개입(Intervene)합니다.
- **결과물 인수**: 완성된 깃 풀리퀘스트(Pull Request)를 검토하고 로컬에 반영합니다.

---

## 5. 고급 팁 및 에이전틱 활용 방법
- **사용자 검증 루프**: Devin이 특정 단계에서 브라우저 테스팅을 완료한 후, 사용자에게 실행 결과 스냅샷을 보여주며 컴포넌트 렌더링에 만족하는지 확인할 수 있습니다. 이때 직접 사용자 가이드라인을 덧붙여 미세조정합니다.
- **타사 라이브러리 연동**: Devin이 한 번도 써보지 않은 최신 AI 라이브러리라도 라이브러리 Github 주소를 던져주면 스스로 문서를 파악하고 즉석에서 예제 코드를 분석해 적용해 냅니다.
