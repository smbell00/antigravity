# Aider 사용자 매뉴얼

## 1. 개요 및 개발사
- **도구명**: Aider (에이더)
- **개발사**: Paul Gauthier 및 오픈소스 커뮤니티
- **한 줄 요약**: 개발자의 로컬 터미널에서 터미널 기반 대화 방식으로 소스 코드를 편집하고 자동으로 깃(Git) 커밋까지 만들어주는 고성능 터미널용 AI 페어 프로그래머입니다.
- **주요 키워드**: `#터미널도구`, `#오픈소스`, `#터미널제어`

---

## 2. 핵심 기능 및 AI 에이전트 기능
- **Git Repository Awareness**: Aider는 깃과 완벽히 연동되어 작동합니다. 변경 사항을 적용한 뒤 의미 있는 커밋 메시지를 AI가 작성하여 자동으로 커밋을 수행합니다.
- **Whole-Project Map**: 대형 리포지토리의 소스 코드 구조를 효율적으로 컴팩트한 맵으로 요약하여 LLM에 최소한의 토큰으로 컨텍스트를 제공하는 전술적 아키텍처를 가집니다.
- **Multi-file editing**: 터미널 명령을 통해 여러 소스 파일을 타겟팅하여 AI가 조화롭게 일괄 편집하도록 설계되었습니다.
- **No-sandbox direct edit**: 사용자의 로컬 환경 디렉토리에서 실시간으로 직접 파일을 수정하므로 샌드박스 업로드 지연 없이 초고속 코딩이 가능합니다.

---

## 3. 설치 및 설정 방법
1. Python 환경에서 `pip`를 통해 쉽게 설치할 수 있습니다:
   ```bash
   pip install aider-chat
   ```
2. OpenAI나 Anthropic API 키를 터미널 환경 변수로 설정합니다:
   ```bash
   export ANTHROPIC_API_KEY=your-api-key
   ```
3. 작업하려는 Git 리포지토리 폴더로 진입한 후 `aider` 명령을 실행합니다.

---

## 4. 핵심 사용 명령어 및 워크플로우
- **`aider` 실행**: 해당 폴더에서 실행하면 AI 채팅 프롬프트가 실행됩니다. 기본적으로 최고의 벤치마크 점수를 내는 Claude 3.5 Sonnet 모델로 시작합니다.
- **`/add <file>`**: AI의 작업 영역(컨텍스트)에 특정 파일을 추가합니다. Aider는 이 파일들을 직접 읽고 수정 권한을 얻습니다.
- **`/drop <file>`**: 더 이상 변경이 필요 없는 파일을 대화 영역에서 제외하여 토큰을 절약합니다.
- **자연어 요청**: `/add src/main.py` 실행 후, "main.py에 예외 처리 로직 추가해줘"라고 입력하면 코드를 즉시 변경하고 자동으로 git commit 합니다.

---

## 5. 고급 팁 및 에이전틱 활용 방법
- **테스트 자동화 피드백 (`/test <command>`)**: 테스트 명령어를 설정하면(예: `/test pytest`), Aider가 소스 코드를 수정한 뒤 자동으로 pytest를 구동합니다. 테스트가 실패할 경우 출력 로그를 읽고 코드가 통과할 때까지 자율적으로 수정 루프를 반복하여 코드를 완성시킵니다.
- **로컬 린트 및 포맷터 연결**: `.aider.conf.yml` 파일을 통하여 코드 포맷팅 도구를 연동해 놓으면, 에이전트가 코드를 변경한 뒤 항상 프로젝트 고유 스타일 가이드대로 자동 다듬어줍니다.
