# GitHub Copilot Workspace 사용자 매뉴얼

## 1. 개요 및 개발사
- **도구명**: GitHub Copilot Workspace (깃허브 코파일럿 워크스페이스)
- **개발사**: GitHub (깃허브) / Microsoft
- **한 줄 요약**: 개발자가 GitHub 이슈(Issue)나 풀 리퀘스트에서 시작하여 계획 수립, 코딩, 테스트 빌드까지 자연어 중심의 에이전트 인터페이스로 수행하는 클라우드 개발 협업 플랫폼입니다.
- **주요 키워드**: `#통합개발환경`, `#상용서비스`, `#깃허브연동`, `#에이전트루프`

---

## 2. 핵심 기능 및 AI 에이전트 기능
- **Task-Centric Flow (작업 중심 워크플로우)**: 코딩을 바로 시작하지 않고, "이슈 -> 브레인스톰/요구사항 정의 -> 구현 계획(Plan) -> 코드 변경(Code) -> 검증 및 풀 리퀘스트" 단계로 설계된 흐름을 따라갑니다.
- **Spec-to-Code**: 이슈에 적힌 한글이나 영문 텍스트를 인지하여 구체적인 기능 명세로 승격시키고 이에 부합하도록 관련 파일들을 일괄 변경합니다.
- **Cloud-based Preview**: 코드가 자동 수정되면 웹 브라우저 내 가상 터미널 환경에서 빌드가 일어나고 바로 실행 화면을 웹뷰 형태로 직접 확인해 볼 수 있습니다.
- **GitHub Codespaces 연동**: 고도화된 수동 편집이나 디버깅이 추가로 필요할 경우 워크스페이스 상태 그대로 Codespaces 컨테이너로 이동해 이어나갈 수 있습니다.

---

## 3. 설치 및 설정 방법
1. 별도의 클라우드 설치 프로그램은 없으며, GitHub 계정에 Copilot Workspace 권한이 활성화되어 있어야 합니다.
2. 사용하고자 하는 깃허브 리포지토리의 이슈 탭으로 이동합니다.
3. 이슈 상세 화면 우측 또는 상단에 나타나는 'Open in Workspace' 버튼을 클릭해 워크스페이스 웹 콘솔로 진입합니다.

---

## 4. 핵심 사용 명령어 및 워크플로우
- **계획 생성 및 승인**: Copilot Workspace가 자동으로 분석한 리포지토리 상태와 해결 계획 목록(Checklist)을 확인한 뒤, 마음에 들지 않는 계획 단계는 체크 해제하거나 자연어로 수정을 지시합니다.
- **코드 생성**: 'Generate Code' 버튼을 클릭하면 AI 에이전트가 각 소스 파일 단위 of Diff(수정 예정 대비 코드)를 나열합니다.
- **로컬 가져오기 또는 PR**: 수정안을 검토한 후 'Create Pull Request'를 눌러 리포지토리에 커밋하거나, GitHub CLI를 통해 로컬 브랜치로 즉시 체크아웃하여 가져옵니다.

---

## 5. 고급 팁 및 에이전틱 활용 방법
- **이슈 구체화 활용**: 이슈에 기능 설계나 체크리스트를 정밀하게 작성할수록 생성되는 구현 계획의 완성도가 기하급수적으로 올라갑니다.
- **동적 빌드 에러 피드백**: 코드가 생성된 후 웹 브라우저상의 테스트 러너에서 에러가 발생하면, 'Fix with Copilot' 버튼을 클릭하여 빌드 로그를 전달하고 AI가 스스로 오류를 디깅하여 다음 패치를 적용하도록 명령할 수 있습니다.
