# Sweep 사용자 매뉴얼

## 1. 개요 및 개발사
- **도구명**: Sweep (스윕)
- **개발사**: Sweep AI 및 오픈소스 커뮤니티
- **한 줄 요약**: 개발팀의 주니어 개발자처럼 작동하는 에이전트로, GitHub 이슈가 생성되면 스스로 코드를 고치고 테스트를 돌려 최종 풀 리퀘스트(PR)를 제출하는 깃허브 네이티브 AI 에이전트입니다.
- **주요 키워드**: `#오픈소스`, `#깃허브연동`, `#에이전트루프`

---

## 2. 핵심 기능 및 AI 에이전트 기능
- **Issue-to-PR Automation**: GitHub Issue에 특정 기능 구현이나 버그 수정을 요청하면 이를 감지하여 연관 소스 코드를 수정하고 브랜치를 만들어 PR을 올리기까지 전 과정을 자율화합니다.
- **Continuous Integration (CI) Feedback Loop**: Sweep이 올린 PR에 대해 CI 빌드(예: GitHub Actions)가 실패하면, 실패한 빌드 로그를 직접 긁어가서 스스로 잘못된 부분을 파악하고 다시 커밋을 추가하는 피드백 루프를 돕습니다.
- **Repository Search (Vector Search)**: 리포지토리 코드를 청크 단위로 나누고 벡터 데이터베이스에 올려 질문에 가장 부합하는 컨텍스트를 고속 탐색합니다.
- **Rules Configuration (`sweep.yaml`)**: 프로젝트 고유 설정을 통해 제외할 파일 패턴, 주로 사용할 프롬프트 팁, 특정 린트 스크립트 실행 지침을 입력받을 수 있습니다.

---

## 3. 설치 및 설정 방법
1. [Sweep AI 공식 페이지](https://sweep.dev) 또는 GitHub Marketplace에서 'Sweep' 앱을 검색합니다.
2. 사용자의 GitHub 계정 및 대상 리포지토리에 Sweep App의 접근 권한을 부여하고 설치를 마칩니다.
3. 프로젝트 루트에 `sweep.yaml` 설정 파일을 생성하여 린트 명령어 및 기본 빌드 구성을 기술합니다.

---

## 4. 핵심 사용 명령어 및 워크플로우
- **이슈 생성**: GitHub Repository에서 새로운 이슈를 생성합니다. 이때 이슈 내용이나 제목에 구체적인 가이드를 제공합니다. (예: "로그인 페이지에 이메일 형식 유효성 검사 로직을 정규식으로 넣어줘.")
- **에이전트 감지**: 이슈가 등록되는 즉시 Sweep이 댓글을 달고 "작업을 시작합니다"라는 로그와 계획을 표시합니다.
- **PR 확인 및 피드백**: 완성된 PR의 변경 파일 차이점(Diff)을 읽어보고 수정사항이 추가로 필요할 경우, 해당 PR 댓글로 자연어 피드백을 적어 올리면 즉각 추가 반영합니다.

---

## 5. 고급 팁 및 에이전틱 활용 방법
- **테스트 연동 디버깅**: `sweep.yaml` 파일에 `run_tests: "npm run test"`와 같이 명시하면 코드를 임의로 바꾼 뒤 로컬 테스트를 알아서 돌려 통과한 버전만 최종적으로 PR로 올립니다. 이로 인해 불안정한 커밋이 마스터 브랜치로 유입되는 것을 완벽 차단합니다.
- **대규모 마이그레이션**: "전체 컴포넌트를 React 18 스타일의 Functional Component로 바꾸기"와 같은 구조적 마이그레이션 태스크를 이슈에 기록하면 수십 개 파일에 걸쳐 안전하게 작업을 수행합니다.
